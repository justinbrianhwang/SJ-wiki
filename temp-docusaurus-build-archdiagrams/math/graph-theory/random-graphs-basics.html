Random Graphs Basics

Random graphs study what a graph typically looks like when edges are chosen by chance. Instead of analyzing one fixed graph, we analyze a probability distribution over many graphs. This shift makes it possible to prove existence results, estimate thresholds for properties, and model large networks whose exact structure is not known in advance.

The basic model is the Erdos-Renyi graph G(n,p)G(n,p)G(n,p): start with nnn labelled vertices and include each possible edge independently with probability ppp. As ppp increases from 000 to 111, the graph evolves from isolated vertices to a dense graph, and properties such as containing a triangle, being connected, or having a giant component appear around predictable thresholds.

Definitions​

The random graph G(n,p)G(n,p)G(n,p) has vertex set [n]={1,…,n}[n]=\{1,\dots,n\}[n]={1,…,n}. Each of the (n2)\binom n2(2n​) possible edges is included independently with probability ppp.

The related model G(n,m)G(n,m)G(n,m) chooses uniformly from all labelled graphs with exactly mmm edges.

For a graph property P\mathcal{P}P, a function p(n)p(n)p(n) is a threshold if the probability that G(n,p)G(n,p)G(n,p) has P\mathcal{P}P changes from near 000 to near 111 around that scale.

A graph property is monotone increasing if adding edges cannot destroy it. Connectivity, containing a triangle, and having a perfect matching are increasing properties.

The expected value of a random variable XXX is denoted E[X]\mathbb{E}[X]E[X]. Indicator variables are the main counting tool: if X=∑iIiX=\sum_i I_iX=∑i​Ii​, then

E[X]=∑iE[Ii].\mathbb{E}[X]=\sum_i \mathbb{E}[I_i].E[X]=i∑​E[Ii​].

Key results​

Expected number of edges. In G(n,p)G(n,p)G(n,p),

E[∣E∣]=p(n2).\mathbb{E}[|E|]=p\binom n2.E[∣E∣]=p(2n​).

Each possible edge contributes 111 with probability ppp.

Expected degree. For any fixed vertex vvv,

E[deg⁡(v)]=(n−1)p.\mathbb{E}[\deg(v)]=(n-1)p.E[deg(v)]=(n−1)p.

The degree has binomial distribution Bin(n−1,p)\mathrm{Bin}(n-1,p)Bin(n−1,p).

Expected triangles. The expected number of triangles is

(n3)p3.\binom n3 p^3.(3n​)p3.

Each triple of vertices forms a triangle exactly when its three edges are present.

Connectivity threshold. The threshold for connectivity in G(n,p)G(n,p)G(n,p) is around

p=log⁡nn.p=\frac{\log n}{n}.p=nlogn​.

Below this scale isolated vertices are likely; above it the graph is likely connected.

Giant component threshold. Around p=1/np=1/np=1/n, a component of linear size emerges.

Linearity of expectation does not require independence. Independence is needed for many probability calculations, but not for expected counts formed by indicators. To count expected triangles, define one indicator for each vertex triple. Even though triangle events overlap and are not independent, the expected total is still the sum of the individual probabilities.

First moment method. If XXX counts bad objects and E[X]→0\mathbb{E}[X]\to 0E[X]→0, then the probability that any bad object exists tends to 000. This follows from Markov's inequality:

Pr⁡(X≥1)≤E[X].\Pr(X\ge 1)\le \mathbb{E}[X].Pr(X≥1)≤E[X].

This method is often used to show that a sparse random graph likely has no copy of a fixed subgraph.

Probabilistic method. If a random graph has positive probability of satisfying a property, then at least one graph with that property exists. This is a nonconstructive existence proof. Ramsey lower bounds, high-girth high-chromatic graphs, and extremal examples are often approached this way.

Almost surely. A property holds asymptotically almost surely if its probability tends to 111 as n→∞n\to\inftyn→∞. Random graph thresholds are asymptotic statements, so finite examples may deviate from the threshold behavior.

Variance and concentration. Expected value gives a center of mass, but it does not by itself say that most samples are close to that value. To prove concentration, one often studies variance or uses inequalities such as Chernoff bounds. For example, the number of edges in G(n,p)G(n,p)G(n,p) is binomial with parameters (n2)\binom n2(2n​) and ppp, so it is sharply concentrated around p(n2)p\binom n2p(2n​) when that expectation is large.

Subgraph thresholds. For a fixed graph HHH, the expected number of labelled copies of HHH in G(n,p)G(n,p)G(n,p) is roughly

n∣V(H)∣p∣E(H)∣n^{|V(H)|}p^{|E(H)|}n∣V(H)∣p∣E(H)∣

up to constants depending on automorphisms. This heuristic predicts the scale at which copies of HHH begin to appear. For triangles, setting n3p3n^3p^3n3p3 near 111 gives p≈1/np\approx 1/np≈1/n.

Random graphs as examples. Random graphs often show that a proposed deterministic statement is false. If a random graph has a property with positive probability, then some concrete graph has that property, even if the proof does not display it. This is the probabilistic method in its most basic form.

Worked example 1: Expected edges and triangles​

Problem. In G(20,0.1)G(20,0.1)G(20,0.1), compute the expected number of edges and expected number of triangles.

Method.

There are

(202)=20⋅192=190\binom{20}{2}=\frac{20\cdot 19}{2}=190(220​)=220⋅19​=190

possible edges. Each appears with probability 0.10.10.1, so

E[∣E∣]=190⋅0.1=19.\mathbb{E}[|E|]=190\cdot 0.1=19.E[∣E∣]=190⋅0.1=19.

For triangles, there are

(203)=20⋅19⋅186=1140\binom{20}{3}=\frac{20\cdot 19\cdot 18}{6}=1140(320​)=620⋅19⋅18​=1140

vertex triples. A fixed triple forms a triangle if all three internal edges appear, which has probability

(0.1)3=0.001.(0.1)^3=0.001.(0.1)3=0.001.

Therefore

E[triangles]=1140⋅0.001=1.14.\mathbb{E}[\text{triangles}]=1140\cdot 0.001=1.14.E[triangles]=1140⋅0.001=1.14.

Checked answer. The expected number of edges is 191919, and the expected number of triangles is 1.141.141.14.

The expectation 1.141.141.14 does not mean the graph must contain a triangle. It means that over many independent samples, the average triangle count approaches 1.141.141.14. Some samples have no triangles, some have one, and occasionally a sample has several overlapping triangles.

Code​

import random
from collections import deque

def random_graph(n, p, seed=None):
    rng = random.Random(seed)
    adj = {i: set() for i in range(n)}
    for i in range(n):
        for j in range(i + 1, n):
            if rng.random() < p:
                adj[i].add(j)
                adj[j].add(i)
    return adj

def component_sizes(adj):
    unseen = set(adj)
    sizes = []
    while unseen:
        start = unseen.pop()
        q = deque([start])
        size = 1
        while q:
            u = q.popleft()
            for v in adj[u]:
                if v in unseen:
                    unseen.remove(v)
                    q.append(v)
                    size += 1
        sizes.append(size)
    return sorted(sizes, reverse=True)

G = random_graph(20, 0.1, seed=7)
print(sum(len(nbrs) for nbrs in G.values()) // 2)
print(component_sizes(G))

Changing the seed changes the sampled graph but not the model. For experiments, it is better to run many samples and average statistics such as edge count, largest component size, number of isolated vertices, or triangle count. One sample can illustrate a definition; many samples reveal the distribution.

A useful simulation habit is to compare empirical averages with the formulas above. For example, repeated samples of G(20,0.1)G(20,0.1)G(20,0.1) should have average edge count close to 191919. If they do not, the most likely error is checking each unordered pair twice or using the wrong probability.

When using random graphs for proofs, state whether the claim is exact, high probability, or positive probability. These are different levels of conclusion. Positive probability proves existence; high probability describes typical behavior as nnn grows.

That wording keeps asymptotic intuition separate from finite numerical claims.

It also makes simulations easier to interpret responsibly.

Random graph calculations should specify the random variable before computing its expectation or probability. "The expected number of triangles" means a sum of indicators over vertex triples; "the probability of a triangle" means at least one triangle exists. These are related but not the same, and confusing them is a common source of incorrect threshold reasoning.

Common pitfalls​

  • Treating G(n,p)G(n,p)G(n,p) as one graph rather than a probability distribution over graphs.
  • Forgetting independence when computing probabilities. Edge events are independent in G(n,p)G(n,p)G(n,p), but many graph properties are not independent.
  • Confusing G(n,p)G(n,p)G(n,p) with G(n,m)G(n,m)G(n,m). One fixes edge probability; the other fixes the exact edge count.
  • Assuming expected value means typical value in every regime. Concentration needs separate proof.
  • Using the connectivity threshold as an exact finite-nnn cutoff. It is an asymptotic statement.
  • Ignoring isolated vertices when reasoning about connectivity near (log⁡n)/n(\log n)/n(logn)/n.
