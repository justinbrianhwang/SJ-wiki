Digraphs Tournaments and Markov Chains

Directed graphs add orientation to adjacency. A road may be one-way, a web page may link to another without receiving a link back, and a state transition may have a preferred direction. Once directions matter, connectedness splits into several notions and matrices become especially useful.

Tournaments and Markov chains are two important directed examples. A tournament orients every edge of a complete graph, modelling pairwise contests. A Markov chain assigns transition probabilities to directed edges, modelling repeated movement between states.

Figure: Transitive tournament on eight vertices. Image: Wikimedia Commons, Booyabazooka, public domain.

Definitions​

A directed graph or digraph DDD has a vertex set V(D)V(D)V(D) and a set of ordered pairs called arcs. An arc from uuu to vvv is written u→vu\to vu→v.

The outdegree d+(v)d^+(v)d+(v) is the number of arcs leaving vvv, and the indegree d−(v)d^-(v)d−(v) is the number of arcs entering vvv. A directed walk follows arc directions. A digraph is strongly connected if for every ordered pair u,vu,vu,v there is a directed path from uuu to vvv. It is weakly connected if the underlying undirected graph is connected.

A tournament is an orientation of a complete graph: for every distinct pair u,vu,vu,v, exactly one of u→vu\to vu→v or v→uv\to uv→u is present.

A finite Markov chain has states and transition probabilities pijp_{ij}pij​, where pijp_{ij}pij​ is the probability of moving from state iii to state jjj in one step. Its transition matrix PPP has nonnegative entries and each row sums to 111.

Key results​

Directed handshaking. In every finite digraph,

∑v∈Vd+(v)=∣A∣=∑v∈Vd−(v),\sum_{v\in V} d^+(v)=|A|=\sum_{v\in V} d^-(v),v∈V∑​d+(v)=∣A∣=v∈V∑​d−(v),

where AAA is the arc set. Each arc contributes once to an outdegree and once to an indegree.

Tournament Hamiltonian path theorem. Every tournament has a directed Hamiltonian path.

Proof sketch: insert vertices one at a time into an existing directed path. Given a path v1→⋯→vkv_1\to \cdots \to v_kv1​→⋯→vk​, place a new vertex xxx before the first viv_ivi​ such that x→vix\to v_ix→vi​, or at the end if no such viv_ivi​ exists. The tournament property guarantees the needed orientations.

Eulerian digraph criterion. A weakly connected finite digraph has a directed Eulerian circuit if and only if

d+(v)=d−(v)d^+(v)=d^-(v)d+(v)=d−(v)

for every vertex vvv.

Markov evolution. If πt\pi_tπt​ is a row vector giving the state distribution at time ttt, then

πt+1=πtP.\pi_{t+1}=\pi_tP.πt+1​=πt​P.

A stationary distribution satisfies π=πP\pi=\pi Pπ=πP.

Strong components. Strong connectedness partitions a digraph into maximal strongly connected subgraphs called strong components. If each strong component is shrunk to one vertex, the resulting condensation digraph is acyclic. This is a major structural difference from undirected components: directed reachability can have one-way movement between components, so the component graph carries order information.

Tournaments and scores. The score of a vertex in a tournament is its outdegree. Scores summarize how many pairwise contests each player wins, but they do not determine the tournament uniquely in general. A vertex with score n−1n-1n−1 beats every other vertex and is a source; a vertex with score 000 loses to every other vertex and is a sink. Strong tournaments have directed paths both ways between every pair of vertices and, in fact, contain directed cycles.

Markov-chain interpretation of digraphs. A transition matrix defines a weighted digraph: draw i→ji\to ji→j when pij>0p_{ij}\gt 0pij​>0. Graph structure then controls probabilistic behavior. If the transition digraph is not strongly connected, probability may flow into a closed class and never return. If it is strongly connected and aperiodic, repeated multiplication often converges to a unique stationary distribution.

Adjacency matrices for digraphs. The adjacency matrix of a digraph has Aij=1A_{ij}=1Aij​=1 when i→ji\to ji→j. It is usually not symmetric. Powers still count directed walks: (Ak)ij(A^k)_{ij}(Ak)ij​ is the number of directed walks of length kkk from iii to jjj. This is the directed analogue of the walk-counting theorem for undirected graphs.

Acyclic digraphs. A directed acyclic graph, or DAG, has no directed cycle. DAGs admit a topological ordering of their vertices, meaning every arc points forward in the order. This is the graph-theoretic model behind prerequisites, task scheduling, and dependency resolution. Tournaments are DAGs exactly when they are transitive tournaments.

Stationary distributions as eigenvectors. With row-vector convention, a stationary distribution satisfies πP=π\pi P=\piπP=π, so it is a left eigenvector of PPP with eigenvalue 111. The entries must be nonnegative and sum to 111, which makes the eigenvector probabilistic rather than merely algebraic.

Worked example 1: Find a Hamiltonian path in a tournament​

Problem. Consider the tournament on vertices {A,B,C,D}\{A,B,C,D\}{A,B,C,D} with arcs

A→B, C→A, A→D, B→C, D→B, C→D.A\to B,\ C\to A,\ A\to D,\ B\to C,\ D\to B,\ C\to D.A→B, C→A, A→D, B→C, D→B, C→D.

Find a directed Hamiltonian path.

Method.

  1. Start with a path containing AAA: simply AAA.
  2. Insert BBB. Since A→BA\to BA→B, the path is A→BA\to BA→B.
  3. Insert CCC. Check against AAA: C→AC\to AC→A, so place CCC before AAA. The path becomes
C→A→B.C\to A\to B.C→A→B.
  1. Insert DDD. Compare DDD with CCC: the arc is C→DC\to DC→D, so DDD cannot go before CCC.
  2. Compare DDD with AAA: the arc is A→DA\to DA→D, so DDD cannot go before AAA.
  3. Compare DDD with BBB: the arc is D→BD\to BD→B, so place DDD before BBB.

The path is

C→A→D→B.C\to A\to D\to B.C→A→D→B.

Check. The arcs C→AC\to AC→A, A→DA\to DA→D, and D→BD\to BD→B are all in the tournament, and every vertex appears exactly once.

Worked example 2: Iterate a Markov chain​

Problem. A weather model has states Sunny SSS and Rainy RRR with transition matrix

P=(0.80.20.40.6).P=\begin{pmatrix} 0.8 & 0.2\\ 0.4 & 0.6 \end{pmatrix}.P=(0.80.4​0.20.6​).

If today is certainly sunny, find the distribution after two days.

Method.

The initial distribution is

π0=(1,0).\pi_0=(1,0).π0​=(1,0).

After one day:

π1=π0P=(1,0)(0.80.20.40.6)=(0.8,0.2).\pi_1=\pi_0P=(1,0) \begin{pmatrix} 0.8 & 0.2\\ 0.4 & 0.6 \end{pmatrix} =(0.8,0.2).π1​=π0​P=(1,0)(0.80.4​0.20.6​)=(0.8,0.2).

After two days:

π2=π1P=(0.8,0.2)(0.80.20.40.6)=(0.8⋅0.8+0.2⋅0.4, 0.8⋅0.2+0.2⋅0.6)=(0.64+0.08, 0.16+0.12)=(0.72,0.28).\begin{aligned} \pi_2 &= \pi_1P\\ &=(0.8,0.2) \begin{pmatrix} 0.8 & 0.2\\ 0.4 & 0.6 \end{pmatrix}\\ &=(0.8\cdot0.8+0.2\cdot0.4,\ 0.8\cdot0.2+0.2\cdot0.6)\\ &=(0.64+0.08,\ 0.16+0.12)\\ &=(0.72,0.28). \end{aligned}π2​​=π1​P=(0.8,0.2)(0.80.4​0.20.6​)=(0.8⋅0.8+0.2⋅0.4, 0.8⋅0.2+0.2⋅0.6)=(0.64+0.08, 0.16+0.12)=(0.72,0.28).​

Checked answer. After two days the model gives probability 0.720.720.72 of sunny and 0.280.280.28 of rainy.

As a row-sum check, 0.72+0.28=10.72+0.28=10.72+0.28=1. Every probability distribution produced by multiplying by a row-stochastic matrix should still sum to 111; if it does not, either the matrix rows were not normalized or the multiplication convention was mixed up.

Code​

def step(dist, P):
    n = len(dist)
    return [sum(dist[i] * P[i][j] for i in range(n)) for j in range(n)]

P = [
    [0.8, 0.2],
    [0.4, 0.6],
]

dist = [1.0, 0.0]
for day in range(1, 6):
    dist = step(dist, P)
    print(day, dist)

def indegree_outdegree(vertices, arcs):
    indeg = {v: 0 for v in vertices}
    outdeg = {v: 0 for v in vertices}
    for u, v in arcs:
        outdeg[u] += 1
        indeg[v] += 1
    return indeg, outdeg

For Markov chains with many states, repeated multiplication is easy to implement but not always the fastest way to find a stationary distribution. Solving the linear system πP=π\pi P=\piπP=π together with ∑iπi=1\sum_i \pi_i=1∑i​πi​=1 is often more direct. Graph structure still matters because reducible chains may have multiple stationary distributions.

For tournament problems, separate existence from construction. The insertion proof gives an explicit Hamiltonian path, but it does not usually give a Hamiltonian cycle. Stronger hypotheses, such as strong connectivity in tournaments, are needed for cycle conclusions.

For Markov-chain calculations, keep the state order fixed from the matrix to every vector. Swapping Sunny and Rainy halfway through a computation produces numbers that may still sum to one but describe the wrong states.

For directed examples, write arrows in every path you claim. A sequence of adjacent vertices in the underlying undirected graph is not necessarily a directed path. For Markov chains, check row sums or column sums according to the convention before doing any long calculation. A valid stochastic matrix should preserve total probability after one step.

Common pitfalls​

  • Treating a directed edge u→vu\to vu→v as if it also allows travel from vvv to uuu.
  • Confusing weak connectedness with strong connectedness.
  • Forgetting that tournaments have exactly one directed arc between every pair of distinct vertices.
  • Using undirected Euler degree conditions on a digraph. Directed Euler circuits require indegree equal to outdegree.
  • Multiplying Markov distributions on the wrong side. Row-vector notation uses πt+1=πtP\pi_{t+1}=\pi_tPπt+1​=πt​P; column-vector notation uses xt+1=PTxtx_{t+1}=P^Tx_txt+1​=PTxt​ or a column-stochastic convention.
  • Accepting a transition matrix whose rows do not sum to 111.
