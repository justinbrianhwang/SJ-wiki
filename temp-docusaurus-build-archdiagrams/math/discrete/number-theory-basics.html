Number Theory Basics

Number theory studies integers and their divisibility structure. In discrete mathematics it supplies proof examples, efficient algorithms, and the arithmetic foundation for cryptography. The central objects are divisibility, primes, greatest common divisors, and integer representations.

The topic is both ancient and computational. Divisibility statements train proof writing; Euclid's algorithm is one of the oldest efficient algorithms; prime factorization leads directly to modern public-key cryptography. The habit to build here is to move between equations, divisibility notation, and algorithms without losing track of hypotheses such as "nonzero" or "positive."

Definitions​

For integers aaa and bbb with a≠0a\ne0a=0, aaa divides bbb, written a∣ba\mid ba∣b, if there is an integer kkk such that b=akb=akb=ak. If no such integer exists, write a∤ba\nmid ba∤b. Divisibility is a statement about exact integer multiples, not approximate division.

The division algorithm says that for every integer aaa and positive integer ddd, there are unique integers qqq and rrr such that

a=dq+r,0≤r<d.a=dq+r,\qquad 0\le r<d.a=dq+r,0≤r<d.

Here qqq is the quotient and rrr is the remainder. This theorem is the basis for modular arithmetic and for Euclid's algorithm.

An integer p>1p\gt 1p>1 is prime if its only positive divisors are 111 and ppp. An integer greater than 111 that is not prime is composite. The number 111 is neither prime nor composite.

The greatest common divisor of integers aaa and bbb, not both zero, is the largest positive integer dividing both. It is written gcd⁡(a,b)\gcd(a,b)gcd(a,b). The least common multiple of nonzero integers aaa and bbb is the smallest positive integer divisible by both, written lcm⁡(a,b)\operatorname{lcm}(a,b)lcm(a,b).

Integers aaa and bbb are relatively prime if gcd⁡(a,b)=1\gcd(a,b)=1gcd(a,b)=1. A base-bbb expansion of a positive integer nnn is an expression

n=akbk+ak−1bk−1+⋯+a1b+a0,n=a_kb^k+a_{k-1}b^{k-1}+\cdots+a_1b+a_0,n=ak​bk+ak−1​bk−1+⋯+a1​b+a0​,

where each digit satisfies 0≤ai<b0\le a_i\lt b0≤ai​<b and ak≠0a_k\ne0ak​=0.

Key results​

Basic divisibility facts:

a∣b∧a∣c  ⟹  a∣(mb+nc)a\mid b \land a\mid c \implies a\mid (mb+nc)a∣b∧a∣c⟹a∣(mb+nc)

for all integers m,nm,nm,n. Proof: if b=asb=asb=as and c=atc=atc=at, then mb+nc=a(ms+nt)mb+nc=a(ms+nt)mb+nc=a(ms+nt), and ms+ntms+ntms+nt is an integer.

The fundamental theorem of arithmetic states that every integer greater than 111 can be written uniquely as a product of primes, up to the order of the factors. This is why prime factorization can be used to compute gcds, lcms, and divisibility.

If nnn is composite, then nnn has a prime divisor at most n\sqrt nn​. Proof: write n=abn=abn=ab with 1<a<n1\lt a\lt n1<a<n and 1<b<n1\lt b\lt n1<b<n. If both aaa and bbb were greater than n\sqrt nn​, then ab>nab\gt nab>n, impossible. So one factor is at most n\sqrt nn​, and that factor has a prime divisor at most itself.

Euclid's algorithm rests on

gcd⁡(a,b)=gcd⁡(b,a mod b).\gcd(a,b)=\gcd(b,a\bmod b).gcd(a,b)=gcd(b,amodb).

If a=bq+ra=bq+ra=bq+r, then a common divisor of aaa and bbb also divides r=a−bqr=a-bqr=a−bq, and a common divisor of bbb and rrr also divides a=bq+ra=bq+ra=bq+r. Thus the common divisors are the same.

There are infinitely many primes. Suppose there were only finitely many, p1,…,pkp_1,\dots,p_kp1​,…,pk​. Let

N=p1p2⋯pk+1.N=p_1p_2\cdots p_k+1.N=p1​p2​⋯pk​+1.

No listed prime divides NNN, because division by any pip_ipi​ leaves remainder 111. But N>1N\gt 1N>1, so by prime factorization it has a prime divisor. That prime is not on the list, contradiction.

For positive integers a,ba,ba,b,

ab=gcd⁡(a,b)lcm⁡(a,b).ab=\gcd(a,b)\operatorname{lcm}(a,b).ab=gcd(a,b)lcm(a,b).

This follows by comparing prime exponents: the gcd takes minimum exponents, the lcm takes maximum exponents, and the sum of the minimum and maximum is the sum of the two original exponents.

Worked example 1: Use the Euclidean algorithm and back-substitution​

Problem. Compute gcd⁡(252,198)\gcd(252,198)gcd(252,198) and express it as an integer combination of 252252252 and 198198198.

Method.

  1. Apply the Euclidean algorithm:
252=198⋅1+54,198=54⋅3+36,54=36⋅1+18,36=18⋅2+0.\begin{aligned} 252 &= 198\cdot1+54,\\ 198 &= 54\cdot3+36,\\ 54 &= 36\cdot1+18,\\ 36 &= 18\cdot2+0. \end{aligned}2521985436​=198⋅1+54,=54⋅3+36,=36⋅1+18,=18⋅2+0.​

Worked example 2: Test primality and factor a number​

Problem. Determine whether 221221221 is prime. If it is composite, factor it.

Method.

  1. Compute the square-root cutoff:
221≈14.86.\sqrt{221}\approx14.86.221​≈14.86.
  1. Test primes at most 14.8614.8614.86: 2,3,5,7,11,132,3,5,7,11,132,3,5,7,11,13.
  2. 221221221 is odd, so 2∤2212\nmid2212∤221.
  3. Sum of digits is 2+2+1=52+2+1=52+2+1=5, so 3∤2213\nmid2213∤221.
  4. It does not end in 000 or 555, so 5∤2215\nmid2215∤221.
  5. Check 777: 7⋅31=2177\cdot31=2177⋅31=217, so 7∤2217\nmid2217∤221.
  6. Check 111111: 11⋅20=22011\cdot20=22011⋅20=220, so 11∤22111\nmid22111∤221.
  7. Check 131313: 13⋅17=22113\cdot17=22113⋅17=221.

Checked answer. 221221221 is composite, and 221=13⋅17221=13\cdot17221=13⋅17. The search could stop as soon as the factor 131313 was found. If no prime up to 221\sqrt{221}221​ had divided it, then 221221221 would have been prime.

Code​

def gcd(a, b):
    a, b = abs(a), abs(b)
    while b:
        a, b = b, a % b
    return a

def factor(n):
    result = []
    d = 2
    while d * d <= n:
        while n % d == 0:
            result.append(d)
            n //= d
        d += 1 if d == 2 else 2
    if n > 1:
        result.append(n)
    return result

def to_base(n, base):
    if n == 0:
        return [0]
    digits = []
    while n:
        n, r = divmod(n, base)
        digits.append(r)
    return digits[::-1]

print(gcd(252, 198))
print(factor(221))
print(to_base(241, 2))

The factor function is trial division. It is suitable for small examples and for learning the logic, but large-scale factoring requires much more sophisticated algorithms.

Common pitfalls​

  • Calling 111 prime. It is neither prime nor composite; otherwise unique factorization would fail.
  • Forgetting the condition a≠0a\ne0a=0 in the definition of a∣ba\mid ba∣b.
  • Using the division algorithm with a negative divisor. The standard statement takes the divisor ddd to be positive.
  • Stopping trial division too late or too early. Testing primes up to n\sqrt nn​ is enough; testing only to n/2n/2n/2 is unnecessary.
  • Confusing gcd with lcm in prime factorizations. Gcd uses minimum exponents; lcm uses maximum exponents.
  • Treating Euclid's algorithm as a black box without preserving the back-substitution equations needed for Bezout coefficients.

Divisibility proofs should always return to the definition. To prove a∣ba\mid ba∣b, exhibit an integer kkk with b=akb=akb=ak. To disprove it, show no such integer can exist, often by using remainders, parity, or bounds. Manipulating the vertical bar as if it were an arithmetic fraction is a common source of invalid reasoning.

The division algorithm is also the reason remainders are well-defined. For a positive divisor ddd, every integer falls into exactly one of the ddd remainder classes 0,1,…,d−10,1,\dots,d-10,1,…,d−1. Negative dividends still have nonnegative remainders in the standard convention: for example, −11=3(−4)+1-11=3(-4)+1−11=3(−4)+1, so the remainder on division by 333 is 111.

Prime factorization gives a second way to compute gcd and lcm. If

a=∏piαi,b=∏piβi,a=\prod p_i^{\alpha_i},\qquad b=\prod p_i^{\beta_i},a=∏piαi​​,b=∏piβi​​,

then

gcd⁡(a,b)=∏pimin⁡(αi,βi),lcm⁡(a,b)=∏pimax⁡(αi,βi).\gcd(a,b)=\prod p_i^{\min(\alpha_i,\beta_i)},\qquad \operatorname{lcm}(a,b)=\prod p_i^{\max(\alpha_i,\beta_i)}.gcd(a,b)=∏pimin(αi​,βi​)​,lcm(a,b)=∏pimax(αi​,βi​)​.

This method is conceptually clear but requires factoring, which may be expensive for large integers. Euclid's algorithm avoids full factorization and is much more efficient for gcd computation.

Base expansion is another application of repeated division. Dividing by the base gives the least significant digit as a remainder; repeating on the quotient gives the next digit. The digits appear in reverse order of discovery, which is why base-conversion algorithms usually collect remainders and then reverse the list.

When doing Euclidean algorithm work by hand, keep the quotient-remainder equations aligned and do not overwrite them. The same equations used to find the gcd are needed for back-substitution. Losing one equation often forces the computation to be repeated from the beginning.

For primality testing, test divisibility only by primes up to the square root, not every integer. If a composite number has a factor larger than n\sqrt nn​, the matching factor is smaller than n\sqrt nn​. This is the reason trial division can stop at the square-root boundary. It is also the reason finding no small prime divisor proves primality.

When using prime factorizations, keep the same prime list for both numbers by allowing exponent 000. For example, 12=22315012=2^2 3^1 5^012=223150 and 20=22305120=2^2 3^0 5^120=223051. Then minimum and maximum exponents can be read componentwise for gcd and lcm without forgetting primes that appear in only one number.
