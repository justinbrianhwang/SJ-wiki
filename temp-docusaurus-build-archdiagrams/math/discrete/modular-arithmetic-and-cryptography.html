Modular Arithmetic and Cryptography

Modular arithmetic treats integers by their remainders. This turns infinite integer questions into finite arithmetic on residue classes, which is why it appears in hashing, pseudorandom generators, checksums, calendars, cyclic schedules, and public-key cryptography.

The main shift is to stop asking for the exact integer and ask only for its class modulo mmm. Once this is done carefully, addition, multiplication, inverses, and exponentiation can be performed with small representatives. Cryptography then uses the fact that some modular operations are easy in one direction but hard to reverse without hidden information.

Definitions​

For integers a,ba,ba,b and a positive integer mmm, aaa is congruent to bbb modulo mmm, written

a≡b(modm),a\equiv b\pmod m,a≡b(modm),

if m∣(a−b)m\mid(a-b)m∣(a−b). Equivalently, aaa and bbb have the same remainder when divided by mmm.

The residue class of aaa modulo mmm is the set

[a]m={b∈Z:b≡a(modm)}.[a]_m=\{b\in\mathbb{Z}:b\equiv a\pmod m\}.[a]m​={b∈Z:b≡a(modm)}.

There are exactly mmm residue classes modulo mmm, represented by 0,1,…,m−10,1,\dots,m-10,1,…,m−1. Arithmetic modulo mmm can be performed on any representatives and then reduced back to this range.

A multiplicative inverse of aaa modulo mmm is an integer bbb such that

ab≡1(modm).ab\equiv1\pmod m.ab≡1(modm).

The inverse, when it exists, is unique modulo mmm. The set of invertible residue classes modulo mmm is often denoted UmU_mUm​.

Euler's phi function ϕ(n)\phi(n)ϕ(n) counts the positive integers not exceeding nnn that are relatively prime to nnn. If n=pqn=pqn=pq for distinct primes ppp and qqq, then

ϕ(n)=(p−1)(q−1).\phi(n)=(p-1)(q-1).ϕ(n)=(p−1)(q−1).

Key results​

Congruence is compatible with addition and multiplication. If

a≡b(modm),c≡d(modm),a\equiv b\pmod m,\qquad c\equiv d\pmod m,a≡b(modm),c≡d(modm),

then

a+c≡b+d(modm),ac≡bd(modm).a+c\equiv b+d\pmod m,\qquad ac\equiv bd\pmod m.a+c≡b+d(modm),ac≡bd(modm).

Proof sketch: m∣(a−b)m\mid(a-b)m∣(a−b) and m∣(c−d)m\mid(c-d)m∣(c−d). Therefore mmm divides (a+c)−(b+d)(a+c)-(b+d)(a+c)−(b+d) and also divides ac−bd=a(c−d)+d(a−b)ac-bd=a(c-d)+d(a-b)ac−bd=a(c−d)+d(a−b).

An inverse of aaa modulo mmm exists if and only if gcd⁡(a,m)=1\gcd(a,m)=1gcd(a,m)=1. This follows from Bezout's identity: there are integers s,ts,ts,t such that

as+mt=1as+mt=1as+mt=1

exactly when gcd⁡(a,m)=1\gcd(a,m)=1gcd(a,m)=1. Reducing modulo mmm gives as≡1(modm)as\equiv1\pmod mas≡1(modm), so sss is an inverse of aaa modulo mmm.

The Chinese remainder theorem says that if m1,…,mkm_1,\dots,m_km1​,…,mk​ are pairwise relatively prime, then the system

x≡ai(modmi)(1≤i≤k)x\equiv a_i\pmod{m_i}\quad(1\le i\le k)x≡ai​(modmi​)(1≤i≤k)

has a unique solution modulo M=m1m2⋯mkM=m_1m_2\cdots m_kM=m1​m2​⋯mk​.

Fast modular exponentiation uses repeated squaring. If the exponent has binary expansion

e=∑jbj2j,e=\sum_j b_j2^j,e=j∑​bj​2j,

then

ae=∏bj=1a2j.a^e=\prod_{b_j=1} a^{2^j}.ae=bj​=1∏​a2j.

Reducing modulo mmm after each multiplication keeps the numbers small and uses only O(log⁡e)O(\log e)O(loge) modular multiplications.

RSA, in simplified mathematical form, chooses primes p,qp,qp,q, sets n=pqn=pqn=pq, chooses eee relatively prime to ϕ(n)\phi(n)ϕ(n), and finds ddd with ed≡1(modϕ(n))ed\equiv1\pmod{\phi(n)}ed≡1(modϕ(n)). Encryption sends MMM to C≡Me(modn)C\equiv M^e\pmod nC≡Me(modn), and decryption computes Cd(modn)C^d\pmod nCd(modn). The practical security rests on the difficulty of factoring large nnn into ppp and qqq.

Worked example 1: Solve a linear congruence​

Problem. Solve

gcd⁡(14,35)=7.\gcd(14,35)=7.gcd(14,35)=7.

Code​

def egcd(a, b):
    if b == 0:
        return (abs(a), 1 if a >= 0 else -1, 0)
    g, x1, y1 = egcd(b, a % b)
    return (g, y1, x1 - (a // b) * y1)

def inv_mod(a, m):
    g, x, _ = egcd(a, m)
    if g != 1:
        raise ValueError("no inverse")
    return x % m

def modexp(base, exp, mod):
    result = 1
    base %= mod
    while exp:
        if exp & 1:
            result = (result * base) % mod
        base = (base * base) % mod
        exp >>= 1
    return result

print(inv_mod(3, 7))
print(modexp(7, 23, 33))

The modexp routine is the repeated-squaring algorithm used in real modular arithmetic libraries, although production cryptographic code must also handle side-channel concerns and padding schemes.

Common pitfalls​

  • Treating a mod m=ba\bmod m=bamodm=b and a≡b(modm)a\equiv b\pmod ma≡b(modm) as the same statement. The first names a remainder; the second is a relation.
  • Cancelling a factor that is not invertible modulo mmm.
  • Forgetting that a linear congruence may have zero, one, or several solutions modulo mmm.
  • Allowing intermediate powers to grow unnecessarily instead of reducing after each multiplication.
  • Thinking RSA security depends on the algorithm being secret. The method is public; the factorization of nnn is the hidden part.
  • Using textbook RSA directly for real messages. Practical cryptography requires padding, randomness, validated parameters, and vetted libraries.

When reducing expressions modulo mmm, reduce early and often. The congruence rules allow replacing a large number by any congruent representative, so 372(mod11)37^2\pmod{11}372(mod11) can become 42(mod11)4^2\pmod{11}42(mod11) and then 16≡5(mod11)16\equiv5\pmod{11}16≡5(mod11). This does not change the residue class, but it keeps arithmetic small and reduces mistakes.

Linear congruences require a gcd check before cancellation. The congruence 6x≡3(mod9)6x\equiv3\pmod96x≡3(mod9) has no solution because gcd⁡(6,9)=3\gcd(6,9)=3gcd(6,9)=3 divides 333, so actually it does have solutions after reducing to 2x≡1(mod3)2x\equiv1\pmod32x≡1(mod3); but 6x≡4(mod9)6x\equiv4\pmod96x≡4(mod9) has none because 3∤43\nmid43∤4. This distinction is exactly why blind division by 666 modulo 999 is invalid.

The Chinese remainder theorem is constructive. To solve x≡a(modm)x\equiv a\pmod mx≡a(modm) and x≡b(modn)x\equiv b\pmod nx≡b(modn) with gcd⁡(m,n)=1\gcd(m,n)=1gcd(m,n)=1, one can search modulo mnmnmn for small examples or build the solution using inverses. The uniqueness is modulo mnmnmn, meaning every solution differs by a multiple of mnmnmn.

In cryptography examples, separate the mathematics from the protocol. Modular exponentiation explains how RSA transforms numbers. Real encryption additionally needs message encoding, padding, randomization, parameter validation, and resistance to timing attacks. A discrete mathematics course usually studies the number-theoretic core, not the full engineering standard.

Finally, distinguish congruence classes from their representatives. The class [3]10[3]_{10}[3]10​ contains …,−7,3,13,23,…\dots,-7,3,13,23,\dots…,−7,3,13,23,…. Writing x≡3(mod10)x\equiv3\pmod{10}x≡3(mod10) identifies a class, not a single integer. If a problem asks for solutions modulo 101010, one representative from each class is enough.

When checking modular computations, substitute the answer into the original congruence rather than only checking the simplified version. If x≡4(mod5)x\equiv4\pmod5x≡4(mod5) came from reducing a congruence modulo 353535, verify several lifted representatives modulo 353535. This catches mistakes in the number of solution classes and in the modulus after division by a gcd.

It is also helpful to keep a table of units for small moduli. Modulo 101010, only 1,3,7,91,3,7,91,3,7,9 have inverses. Modulo a prime ppp, every nonzero residue has an inverse. This difference explains why fields such as Zp\mathbb{Z}_pZp​ behave much more like ordinary arithmetic than composite-modulus systems such as Z10\mathbb{Z}_{10}Z10​.

For worked cryptography exercises, use intentionally small primes only to understand the mechanism. Small RSA examples are insecure precisely because factoring nnn is easy. The lesson is the algebraic loop: choose relatively prime exponents, use an inverse modulo ϕ(n)\phi(n)ϕ(n), and rely on modular exponentiation for encryption and decryption. Do not infer practical parameter sizes or security practices from classroom-sized numbers.

In modular arithmetic, the modulus is part of the statement. The claim x≡2x\equiv2x≡2 is incomplete until the modulus is named. Changing the modulus changes the residue classes, the invertible elements, and the number of distinct solutions.

For that reason, carry the modulus through every displayed step instead of mentioning it only at the beginning.

This habit makes invalid cancellation easier to notice.
