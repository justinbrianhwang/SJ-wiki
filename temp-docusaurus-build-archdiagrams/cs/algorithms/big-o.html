Big-O Notation

Big-O notation describes the asymptotic upper bound of a function's growth rate. It's how we compare algorithm efficiency without caring about constants or hardware.

Formal definition​

We say f(n)=O(g(n))f(n) = O(g(n))f(n)=O(g(n)) if there exist positive constants ccc and n0n_0n0​ such that:

0≤f(n)≤c⋅g(n)for all n≥n00 \le f(n) \le c \cdot g(n) \quad \text{for all } n \ge n_00≤f(n)≤c⋅g(n)for all n≥n0​

In plain English: fff grows at most as fast as ggg, up to a constant factor, for sufficiently large nnn.

Example: linear search​

def linear_search(arr, target):
    for i, x in enumerate(arr):
        if x == target:
            return i
    return -1

The loop runs at most nnn times, so this is O(n)O(n)O(n).

Example: binary search​

def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1

Each iteration halves the search space, so it takes O(log⁡n)O(\log n)O(logn) time.

warning

Big-O describes the worst case. Big-Θ (theta) describes a tight bound; Big-Ω (omega) describes a lower bound. Don't conflate them.
