Attacks on LLMs and Other Modalities

Adversarial ML began with image classifiers in much of the modern deep-learning literature, but the same security questions appear in text, speech, reinforcement learning, recommender systems, multimodal models, and LLM applications. The perturbation set changes: a text edit is discrete and semantic, an audio perturbation passes through acoustics, an RL perturbation can change a trajectory, and an LLM prompt injection may be an instruction placed in retrieved content or tool output.

This page gives the vocabulary needed to connect image adversarial examples to other modalities without pretending that every domain fits an ℓp\ell_pℓp​ ball. Representative papers such as HotFlip, TextFooler, BERT-Attack, audio attacks, and RF universal perturbations are folded into the topical modality sections below.

Definitions​

A text adversarial example changes a text input so that a model changes its prediction while the meaning, fluency, or label should remain stable. The perturbation set may include character edits, word substitutions, paraphrases, syntax changes, or distractor sentences.

For a text classifier fff, an untargeted attack can be written abstractly as:

max⁡x′∈N(x)L(f(x′),y),\max_{x' \in \mathcal{N}(x)} \mathcal{L}(f(x'), y),x′∈N(x)max​L(f(x′),y),

where N(x)\mathcal{N}(x)N(x) is a discrete neighborhood of valid edits. Unlike image pixels, tokens are not continuous input coordinates once tokenization is fixed. Gradient methods may operate on embeddings or one-hot token representations, but the final attack must map back to valid text.

HotFlip-style attacks use gradients with respect to one-hot token indicators to score discrete substitutions. If replacing token aaa with token bbb changes a one-hot vector by eb−eae_b-e_aeb​−ea​, a first-order score is:

ΔL≈∇xL⊤(eb−ea).\Delta \mathcal{L} \approx \nabla_x \mathcal{L}^\top (e_b-e_a).ΔL≈∇x​L⊤(eb​−ea​).

TextFooler-style attacks search for important words and replace them with semantically similar alternatives while checking classifier output and semantic similarity constraints.

An audio adversarial example perturbs a waveform so an automatic speech recognizer or audio classifier produces a target transcript or label. The constraint may involve ℓp\ell_pℓp​ size, signal-to-noise ratio, psychoacoustic masking, playback robustness, or room acoustics.

An RL adversarial attack perturbs observations, rewards, dynamics, or policies so an agent chooses poor actions. Because decisions affect future states, a small perturbation at one time can compound over a trajectory.

A jailbreak attempts to make an LLM produce behavior disallowed by its instruction hierarchy or safety policy. A prompt injection places instructions in user content, retrieved documents, web pages, tool outputs, or other context so the model follows untrusted instructions over trusted ones. These are not usually norm-bounded perturbations; they are instruction and context attacks.

Key results​

The main lesson across modalities is that validity constraints are domain-specific. For images, ℓ∞\ell_\inftyℓ∞​ and ℓ2\ell_2ℓ2​ budgets are convenient approximations. For text, preserving label and meaning is a semantic constraint. For audio, imperceptibility and over-the-air robustness depend on human hearing and the playback-recording channel. For RL, the attack objective may be cumulative reward rather than one-step classification loss. For LLM systems, the target is often policy compliance, tool misuse, data exfiltration, or instruction hierarchy failure.

Discrete domains make optimization harder. A token substitution is not an infinitesimal pixel change, and tokenization can make small character edits produce large representation changes. Gradient methods can still help by scoring candidate edits, but search, language-model constraints, semantic similarity models, and human evaluation are often needed.

LLM-era attacks broaden the threat model from model weights to systems. A deployed LLM application may include:

  • A system prompt and developer instructions.
  • User messages.
  • Retrieved documents.
  • Tool calls and tool outputs.
  • Memory, files, browser content, or email content.
  • Post-processing filters or policy classifiers.

Prompt injection exploits the fact that untrusted text can be placed in the model context next to trusted instructions. The attack surface is therefore not only the model but also retrieval, tool routing, content isolation, output parsing, and permission boundaries. Robustness requires system design, not just model fine-tuning.

Multimodal models combine risks. An image can contain text instructions, a document can contain hidden or low-contrast instructions, and a web page can mix ordinary content with instructions meant for an agent. The core threat-model question remains the same: what input channels can the attacker control, and what behavior counts as success?

Representative modality-specific attacks​

Gradient-scored character edits​

Ebrahimi et al. [1] introduced HotFlip as a white-box text attack that scores discrete character edits using gradients. It treats a substitution from current character aaa to candidate character bbb at position iii as a one-hot change:

Δxi,a→b=ei,b−ei,a.\Delta x_{i,a\to b}=e_{i,b}-e_{i,a}.Δxi,a→b​=ei,b​−ei,a​.

The first-order loss change is:

L(x+Δx)−L(x)≈∇xL(x)⊤Δx,\mathcal{L}(x+\Delta x)-\mathcal{L}(x) \approx \nabla_x\mathcal{L}(x)^\top\Delta x,L(x+Δx)−L(x)≈∇x​L(x)⊤Δx,

so the substitution score is:

s(i,a,b)=∂L∂xi,b−∂L∂xi,a.s(i,a,b)= \frac{\partial\mathcal{L}}{\partial x_{i,b}} - \frac{\partial\mathcal{L}}{\partial x_{i,a}}.s(i,a,b)=∂xi,b​∂L​−∂xi,a​∂L​.

Worked micro-example: if the current character is a, the candidate is b, and the gradient entries are −0.4-0.4−0.4 and 0.70.70.7, then s=0.7−(−0.4)=1.1s=0.7-(-0.4)=1.1s=0.7−(−0.4)=1.1. For an untargeted attack, that is a strong candidate because it is estimated to increase loss.

HotFlip is the discrete analogue of a first-order attack, but validity is linguistic. The page or experiment must define the allowed alphabet, edit types, edit budget, tokenization, and semantic constraints.

Semantics-constrained word substitutions​

Jin et al. [2] introduced TextFooler as a black-box word-substitution attack for text classification and entailment. Its contribution was a practical pipeline: rank important words, propose semantically similar substitutions, filter candidates, and query the victim model until the prediction changes.

A common importance score masks or removes word iii and measures true-label confidence drop:

Ii=py(x)−py(x∖i).I_i=p_y(x)-p_y(x_{\setminus i}).Ii​=py​(x)−py​(x∖i​).

The attack enforces constraints such as:

sim(x,x′)≥τ,#changed words#words≤ρ.\mathrm{sim}(x,x')\ge\tau, \qquad \frac{\#\mathrm{changed\ words}}{\#\mathrm{words}}\le\rho.sim(x,x′)≥τ,#words#changed words​≤ρ.

Worked micro-example: if masking "excellent" drops true-class probability from 0.920.920.92 to 0.410.410.41, its importance score is 0.510.510.51. If masking "movie" only drops the score to 0.860.860.86, its score is 0.060.060.06, so "excellent" is tried first.

TextFooler-style success is only meaningful when the modified sentence keeps the original human label. A label change caused by changing a negation, entity, date, or sentiment word is not an adversarial success under a label-preserving threat model.

Contextual masked-language-model substitutions​

Li et al. [3] introduced BERT-Attack, which uses a masked language model to propose context-aware replacement candidates. The victim can remain black-box: the masked LM generates candidates, filters remove invalid substitutions, and the victim classifier is queried to select a damaging replacement.

For an important word wiw_iwi​, the attack masks the position and draws candidate replacements from:

c1,…,cK∼pMLM(wi∣xmask(i)).c_1,\dots,c_K \sim p_{\mathrm{MLM}}(w_i\mid x_{\mathrm{mask}(i)}).c1​,…,cK​∼pMLM​(wi​∣xmask(i)​).

Then it accepts a valid candidate that causes misclassification or most reduces true-label confidence.

Compact pseudo-code:

rank words by victim confidence drop
for word in ranked_words:
    candidates = masked_language_model(mask(sentence, word))
    candidates = semantic_and_syntax_filter(candidates)
    query victim on candidates
    keep the first valid success or best confidence drop

The generator checkpoint, top-KKK candidate budget, tokenizer rules, semantic threshold, and victim-query count are all part of the attack. Masked-LM plausibility is not the same thing as label preservation.

Targeted waveform attacks​

Carlini and Wagner [4] adapted optimization-based adversarial attacks to automatic speech recognition by optimizing a waveform perturbation against a sequence loss. For target transcript lll, a digital CTC-based attack can solve:

min⁡δ∥δ∥22+c LCTC(f(x+δ),l)subject tox+δ∈[−1,1]T.\min_\delta \|\delta\|_2^2 +c\,\mathcal{L}_{\mathrm{CTC}}(f(x+\delta),l) \quad\text{subject to}\quad x+\delta\in[-1,1]^T.δmin​∥δ∥22​+cLCTC​(f(x+δ),l)subject tox+δ∈[−1,1]T.

The CTC term handles unknown alignment between audio frames and transcript characters:

LCTC=−log⁡p(l∣x+δ).\mathcal{L}_{\mathrm{CTC}}=-\log p(l\mid x+\delta).LCTC​=−logp(l∣x+δ).

Worked micro-example: if the target transcript probability rises from 0.0010.0010.001 to 0.250.250.25, the CTC negative log-likelihood falls from −log⁡(0.001)=6.9078-\log(0.001)=6.9078−log(0.001)=6.9078 to −log⁡(0.25)=1.3863-\log(0.25)=1.3863−log(0.25)=1.3863, a decrease of about 5.525.525.52.

Audio validity cannot be reduced to waveform ℓ2\ell_2ℓ2​ alone. Reports should specify sample rate, clipping, decoder, transcript matching, SNR or perceptual constraints, and whether the attack is digital or over-the-air through speakers and microphones.

Radio-frequency universal perturbations​

Wang et al. [5] adapted universal perturbation ideas to automatic modulation classification. The perturbation is added to in-phase and quadrature signal samples rather than pixels, and the constraint set comes from communications engineering: power, frequency content, shift, channel, and data availability.

A generic universal RF objective is:

max⁡vE(x,y)∼D[L(f(x+v),y)]subject tov∈CRF.\max_v \mathbb{E}_{(x,y)\sim\mathcal{D}} \left[ \mathcal{L}(f(x+v),y) \right] \quad\text{subject to}\quad v\in\mathcal{C}_{\mathrm{RF}}.vmax​E(x,y)∼D​[L(f(x+v),y)]subject tov∈CRF​.

For a simple power budget ∥v∥22≤P\|v\|_2^2\le P∥v∥22​≤P, projection scales by:

vproj=v⋅min⁡(1,P∥v∥2).v_{\mathrm{proj}} = v\cdot\min\left(1,\frac{\sqrt{P}}{\|v\|_2}\right).vproj​=v⋅min(1,∥v∥2​P​​).

Worked micro-example: if ∥v∥22=25\|v\|_2^2=25∥v∥22​=25 and the allowed energy is P=9P=9P=9, then ∥v∥2=5\|v\|_2=5∥v∥2​=5, P=3\sqrt{P}=3P​=3, and the scale factor is 3/5=0.63/5=0.63/5=0.6.

RF attacks should be reported with signal representation, sample rate, SNR distribution, power and spectral constraints, channel assumptions, and whether the perturbation was digitally injected, replayed, or transmitted.

Worked example 1: Word-substitution attack budget​

Problem: A sentiment classifier processes a 20-token sentence. An attack changes 3 tokens using synonym substitutions that preserve the human label. Compute the token edit fraction and explain why this is not the same as an ℓ∞\ell_\inftyℓ∞​ image budget.

  1. Total tokens:
n=20.n=20.n=20.
  1. Edited tokens:
k=3.k=3.k=3.
  1. Edit fraction:
kn=320=0.15.\frac{k}{n}=\frac{3}{20}=0.15.nk​=203​=0.15.
  1. Convert to percent:
0.15⋅100%=15%.0.15 \cdot 100\% = 15\%.0.15⋅100%=15%.
  1. In images, an ℓ∞\ell_\inftyℓ∞​ budget limits every coordinate by a small continuous amount. In text, a single token replacement can be a large discrete jump in embedding space but small in meaning, or it can completely change the label.

Checked answer: the edit fraction is 15%15\%15%. The attack must be judged by semantic and grammatical validity, not by direct analogy to pixel-wise ϵ\epsilonϵ.

Worked example 2: Prompt-injection threat modeling​

Problem: An LLM assistant summarizes web pages using a browsing tool. The web page contains untrusted text that asks the assistant to ignore prior instructions and reveal private notes. Classify the attack surface and state the correct robustness question without writing an attack prompt.

  1. The attacker's capability is control over retrieved page content:
C=write untrusted text that enters model context.\mathcal{C}=\text{write untrusted text that enters model context}.C=write untrusted text that enters model context.
  1. The attacker's goal is not image misclassification. It is instruction-hierarchy violation or data exfiltration:
G=make the assistant follow untrusted page instructions.\mathcal{G}=\text{make the assistant follow untrusted page instructions}.G=make the assistant follow untrusted page instructions.

  1. The perturbation set is not an ℓp\ell_pℓp​ ball. It is a set of possible page contents, layouts, hidden text, or quoted instructions.

  2. The correct defense question is:

Does the system keep untrusted content from overriding trusted instructions or permissions?\text{Does the system keep untrusted content from overriding trusted instructions or permissions?}Does the system keep untrusted content from overriding trusted instructions or permissions?
  1. A good evaluation would include tool-output isolation, retrieval labeling, refusal behavior, permission checks, and tests where untrusted content conflicts with user or system instructions.

Checked answer: this is a prompt-injection threat model against an LLM system, not a norm-bounded adversarial example. Robustness depends on instruction hierarchy and system boundaries.

Code​

import torch

def hotflip_scores(embedding_grad, embedding_matrix, token_id):
    # embedding_grad: gradient of loss with respect to the current token embedding, shape [dim]
    # embedding_matrix: candidate token embeddings, shape [vocab, dim]
    # First-order score for replacing current token a with candidate b:
    # grad^T (emb[b] - emb[a])
    current = embedding_matrix[token_id]
    delta = embedding_matrix - current
    return torch.mv(delta, embedding_grad)

def top_replacement_candidates(embedding_grad, embedding_matrix, token_id, banned_ids, k=10):
    scores = hotflip_scores(embedding_grad, embedding_matrix, token_id)
    scores[banned_ids] = -float("inf")
    scores[token_id] = -float("inf")
    values, indices = torch.topk(scores, k)
    return list(zip(indices.tolist(), values.tolist()))

This is a gradient-based candidate scorer for discrete text substitutions. It does not decide whether a replacement preserves meaning, grammar, or label; a real text attack must add those constraints and should be evaluated with human or high-quality semantic checks.

Common pitfalls​

  • Forcing every modality into image-style ℓp\ell_pℓp​ notation when the validity constraint is semantic, acoustic, temporal, or procedural.
  • Counting text attacks as successful when they change the true human label.
  • Ignoring tokenization effects, such as a small character edit producing very different token sequences.
  • Evaluating audio attacks only digitally when the claim is over-the-air robustness.
  • Measuring RL attack success with one-step action changes instead of cumulative reward or safety outcomes.
  • Describing LLM jailbreaks without separating model behavior from application-level tool permissions and context isolation.
  • Treating prompt injection as solved by output filtering alone; the system must manage untrusted instructions before tool use and data access.
