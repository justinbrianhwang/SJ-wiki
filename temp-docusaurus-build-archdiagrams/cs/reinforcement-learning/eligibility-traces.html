Eligibility Traces

Eligibility traces provide a bridge between one-step TD methods and Monte Carlo methods. They let recent states or state-action pairs remain eligible for credit when later TD errors arrive. The forward view describes the target as a weighted mixture of n-step returns. The backward view gives an online mechanism: maintain a trace vector, update it each step, and use the current TD error to adjust all recently eligible predictions.

Sutton and Barto use eligibility traces to unify many earlier ideas. TD(λ\lambdaλ), SARSA(λ\lambdaλ), true online TD(λ\lambdaλ), Watkins's Q(λ\lambdaλ), and tree-backup traces differ in how traces are accumulated, cleared, or weighted, but they share the idea that credit assignment should extend over time without waiting for a full episode.

Definitions​

The λ\lambdaλ-return is a geometrically weighted average of n-step returns:

Gtλ=(1−λ)∑n=1∞λn−1Gt:t+nG_t^\lambda = (1-\lambda)\sum_{n=1}^{\infty}\lambda^{n-1}G_{t:t+n}Gtλ​=(1−λ)n=1∑∞​λn−1Gt:t+n​

in the continuing case, with appropriate truncation in episodic tasks. The parameter 0≤λ≤10\le\lambda\le10≤λ≤1 controls how much weight goes to longer returns. λ=0\lambda=0λ=0 gives the one-step TD target. λ=1\lambda=1λ=1 approaches the Monte Carlo return in episodic tasks.

For linear value prediction, the accumulating eligibility trace is

zt=γλzt−1+∇v^(St,wt).\mathbf{z}_t = \gamma\lambda \mathbf{z}_{t-1}+\nabla \hat v(S_t,\mathbf{w}_t).zt​=γλzt−1​+∇v^(St​,wt​).

The TD(λ\lambdaλ) update is

wt+1=wt+αδtzt,\mathbf{w}_{t+1} = \mathbf{w}_t + \alpha\delta_t\mathbf{z}_t,wt+1​=wt​+αδt​zt​,

where

δt=Rt+1+γv^(St+1,wt)−v^(St,wt).\delta_t=R_{t+1}+\gamma\hat v(S_{t+1},\mathbf{w}_t)-\hat v(S_t,\mathbf{w}_t).δt​=Rt+1​+γv^(St+1​,wt​)−v^(St​,wt​).

For tabular state values, ∇v^(St,wt)\nabla \hat v(S_t,\mathbf{w}_t)∇v^(St​,wt​) is a one-hot vector, so the trace for the current state increases while previous traces decay by γλ\gamma\lambdaγλ.

SARSA(λ\lambdaλ) uses traces over state-action pairs:

zt(s,a)=γλzt−1(s,a)+1s=St,a=At.z_t(s,a)=\gamma\lambda z_{t-1}(s,a)+\mathbb{1}_{s=S_t,a=A_t}.zt​(s,a)=γλzt−1​(s,a)+1s=St​,a=At​​.

Watkins's Q(λ\lambdaλ) is an off-policy control method that cuts traces when a nongreedy action is selected, preserving a connection to Q-learning's greedy target.

True online TD(λ\lambdaλ) modifies the backward view so that it more exactly matches the online forward view at every time step, not only approximately or at episode end.

Key results​

Eligibility traces solve a practical credit-assignment problem. One-step TD propagates information one step per update. Monte Carlo propagates complete returns but waits until episode end and can have high variance. TD(λ\lambdaλ) can propagate a TD error backward through many recent states immediately, with influence decaying according to γλ\gamma\lambdaγλ.

The forward view is conceptually clean: update toward GtλG_t^\lambdaGtλ​. The backward view is computationally useful: update with δtzt\delta_t\mathbf{z}_tδt​zt​. Under appropriate conditions, these views are equivalent for offline updates, and true online TD(λ\lambdaλ) sharpens the equivalence for online learning with changing weights.

The trace parameter λ\lambdaλ is not just "memory length." The effective decay is γλ\gamma\lambdaγλ, so discounting also shortens credit assignment. With γ=0.9\gamma=0.9γ=0.9 and λ=0.8\lambda=0.8λ=0.8, a trace decays by 0.720.720.72 each step before any new activation.

Accumulating traces can exceed 111 for repeatedly visited states or features. Replacing traces set the active trace to 111 instead of adding 111, which can improve behavior in some tabular tasks by preventing repeated visits from producing very large traces. With function approximation, trace variants require more care.

Control with traces must respect the policy relationship. On-policy SARSA(λ\lambdaλ) can keep traces through exploratory actions because it evaluates the exploratory policy. Watkins's Q(λ\lambdaλ) cuts traces after nongreedy actions because Q-learning's target is greedy, not the exploratory behavior.

Eligibility traces can be understood as a memory of responsibility. A state visited several steps ago may still deserve part of the current TD error if its trace has not decayed away. This does not mean the old state caused the new reward in a philosophical sense; it means the update rule assigns credit according to recency, discounting, and the chosen trace parameter. The trace is therefore an algorithmic credit-assignment device, not an additional environment model.

In function approximation, the trace vector lives in parameter space rather than as a table over states. If v^(s,w)\hat v(s,\mathbf{w})v^(s,w) is linear, the trace accumulates feature vectors. If the approximator is nonlinear, the trace accumulates gradients. This makes traces compatible with approximation in principle, but it also means that feature scaling and changing gradients affect how credit is distributed.

True online TD(λ\lambdaλ) is important because ordinary accumulating-trace TD can diverge from the intended online forward view as weights change during an episode. The true-online correction keeps the computational advantages of the backward view while matching the forward-view target more faithfully at each step.

Code​

import numpy as np

rng = np.random.default_rng(9)
n_states = 7
V = np.zeros(n_states)
alpha, gamma, lam = 0.1, 1.0, 0.8

def episode():
    s = 3
    states = [s]
    rewards = []
    while 0 < s < 6:
        s += rng.choice([-1, 1])
        rewards.append(1.0 if s == 6 else 0.0)
        states.append(s)
    return states, rewards

for _ in range(1000):
    z = np.zeros(n_states)
    states, rewards = episode()
    for t, r in enumerate(rewards):
        s = states[t]
        sp = states[t + 1]
        z *= gamma * lam
        z[s] += 1.0
        target_next = 0.0 if sp in (0, 6) else V[sp]
        delta = r + gamma * target_next - V[s]
        V += alpha * delta * z
        V[0] = V[6] = 0.0

print(np.round(V[1:6], 3))

Common pitfalls​

  • Treating λ\lambdaλ as the only decay factor. Traces decay by γλ\gamma\lambdaγλ, not just λ\lambdaλ.
  • Forgetting to reset traces at episode boundaries.
  • Mixing forward-view and backward-view formulas without checking whether the algorithm is offline, online, accumulating, replacing, or true online.
  • Letting traces grow unintentionally in loops with accumulating traces. Replacing traces may be more appropriate in some tabular control tasks.
  • Failing to cut traces in Watkins's Q(λ\lambdaλ) after nongreedy actions.
  • Assuming traces remove the need for exploration. They improve credit assignment after experience occurs; they do not guarantee useful experience.
